# TypeScript 학습 규칙 및 컨벤션

### 📂 파일 관리 규칙
- 새로운 개념을 공부할 때마다 `TypeScript/` 폴더 내에 Markdown 파일로 기록합니다.
- 파일 이름은 언더바(`_`)나 대시(`-`) 대신 **공백(Space)**을 사용하며, 제목은 **한글**로 작성합니다.
  - 개념 학습 문서의 경우 앞에 넘버링을 붙여 정렬할 수 있습니다. (예시: `01 기본 개념 및 장점.md`)
  - 환경 설정이나 실행 방법 같은 공통 가이드 문서의 경우 넘버링 없이 작성할 수 있습니다. (예시: `타입스크립트 실행.md`)
- `README.md` 파일은 메인 대문 역할을 하므로 개별 개념 정리 시 수정하지 않고 그대로 유지합니다.


### 📝 문서 작성 스타일
- 마크다운 문법을 활용하여 시각적으로 깔끔하고 가독성 있게 정리합니다.
- 개념은 생략 없이 **최대한 자세하게** 기록하며, 깊이 있는 이해를 돕도록 작성합니다.
- 코드 예제는 가능한 한 TypeScript 컴파일 에러 예시와 올바른 사용 예시를 비교하여 작성합니다.
- 핵심 개념은 **굵은 글씨**나 `하이라이트`를 적극 활용합니다.

### 🚀 Git 작업 프로세스
- 개념 정리가 완료된 후, **유저가 명시적으로 커밋과 푸쉬를 요청할 때만** Git 작업을 수행합니다. (정리 직후 자동으로 커밋/푸쉬하지 않습니다.)
- 커밋 메시지는 다음과 같은 규칙을 따릅니다:
  - 문서 추가/수정: `docs: add [개념 이름] note` 또는 `docs: update [개념 이름] note`
