# TypeScript 학습 규칙 및 컨벤션

### 📂 파일 관리 규칙
- 새로운 개념을 공부할 때마다 `TypeScript/` 폴더 내에 넘버링을 붙여 Markdown 파일로 기록합니다.
- 파일 이름은 언더바(`_`) 대신 **공백(Space)**을 사용하며, 제목은 **한글**로 작성합니다.
  - 예시: `01 기본 개념 및 장점.md`, `02 기본 타입 정리.md`
- `README.md` 파일은 메인 대문 역할을 하므로 개별 개념 정리 시 수정하지 않고 그대로 유지합니다.

### 📝 문서 작성 스타일
- 마크다운 문법을 활용하여 가독성 있게 정리합니다.
- 코드 예제는 가능한 한 TypeScript 컴파일 에러 예시와 올바른 사용 예시를 비교하여 작성합니다.
- 핵심 개념은 **굵은 글씨**나 `하이라이트`를 적극 활용합니다.

### 🚀 Git 작업 프로세스
- 하나의 개념 정리가 끝날 때마다 독립적으로 커밋(Commit)하고 원격 저장소(`main` 브랜치)에 푸쉬(Push)합니다.
- 커밋 메시지는 다음과 같은 규칙을 따릅니다:
  - 문서 추가/수정: `docs: add [개념 이름] note` 또는 `docs: update [개념 이름] note`
