# 타입스크립트 실행

## 실행 방법

### 컴파일 후 실행

```bash
npx tsc && node dist/index.js
```

TypeScript를 JavaScript로 컴파일한 뒤 Node.js로 실행한다.

### ts-node로 바로 실행

```bash
npx ts-node src/index.ts
```

### tsx로 바로 실행

```bash
npx tsx src/index.ts
```

---

## 도구 비교

| | ts-node | tsx |
|---|---|---|
| 줄임말 | TypeScript Execute (비공식) | TypeScript Execute (비공식) |
| 속도 | 느림 | 빠름 |
| 타입 체크 | O | X |
| 내부 엔진 | tsc | esbuild |

---

## 타입 체크만 하기

```bash
npx tsc --noEmit
```

`.js` 파일을 생성하지 않고 타입 오류만 확인한다.  
`tsx`는 타입 체크를 건너뛰기 때문에, 타입 오류 확인이 필요할 때 따로 실행한다.

---

## 관련 용어

- **npx** — Node Package eXecute. 패키지를 설치 없이 바로 다운로드해서 실행해주는 도구.
- **tsc** — TypeScript Compiler. `.ts` 파일을 `.js`로 컴파일한다.
- **tsx** — TypeScript eXecute (비공식). esbuild 기반으로 타입스크립트를 빠르게 실행한다.
