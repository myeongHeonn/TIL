# 01. TypeScript 기본 개념 및 장점

### 1. TypeScript의 기본 개념
TypeScript는 **JavaScript에 강력한 타입(Type) 시스템을 더한 자바스크립트의 상위 확장(Superset) 언어**입니다. 마이크로소프트에서 개발했으며, 결국에는 일반 자바스크립트 코드로 컴파일(변환)되어 브라우저나 Node.js 환경에서 실행됩니다.

*   **정적 타입 지원:** 코드를 실행하기 전(컴파일 시점)에 데이터의 타입을 명시하고 검사합니다.
*   **컴파일 언어:** 브라우저는 TypeScript를 이해하지 못하므로, `tsc`라는 컴파일러를 통해 자바스크립트 코드로 변환하는 과정이 필요합니다.

---

### 2. TypeScript의 핵심 장점

#### ① 에러 사전 방지
자바스크립트는 코드가 실제로 실행되어야만 오류를 발견할 수 있는 동적 타입 언어입니다. 반면 타입스크립트는 **코드를 작성하는 도중(컴파일 단계)에 타입 오류를 찾아내어 대규모 버그를 사전에 차단**합니다.

```typescript
function sum(a: number, b: number) {
  return a + b;
}
// 컴파일 에러 발생: 'string' 형식의 인수는 'number' 형식의 매개 변수에 할당될 수 없습니다.
sum('5', 10); 
```

#### ② 최상의 개발자 경험 (IDE 자동 완성)
코드 편집기가 변수나 함수의 타입을 정확히 알고 있기 때문에, 오타를 줄여주고 **강력한 자동 완성(IntelliSense)**과 함수 매개변수 가이드를 제공합니다.

#### ③ 안전하고 쉬운 리팩토링
규모가 큰 프로젝트에서 변수명이나 함수의 구조를 바꿀 때, 타입스크립트는 추적 시스템이 완벽하여 **이름 변경(Rename) 기능 한 번으로 프로젝트 전체의 참조를 안전하게 변경**할 수 있습니다.

#### ④ 강력한 문서화 역할
코드 자체가 움직이는 문서(Document) 역할을 합니다. 함수의 입력값과 출력값의 타입이 명확히 명시되어 있기 때문에, 코드의 의도를 파악하기 훨씬 쉽습니다.
