# 변수와 데이터 타입

## 개요

TypeScript에서 변수를 선언하고 타입을 지정하는 방식을 이해하는 것은 안정적이고 오류 없는 코드를 작성하기 위한 기초입니다. 이 장에서는 변수 선언 방식, TypeScript의 기본 데이터 타입, 그리고 타입 어노테이션과 추론을 학습합니다.

---

## 1. 변수 선언 (var, let, const)

### 1-1. const (권장, 기본값)

`const`는 **재할당이 불가능한 변수**를 선언합니다. 값이 변경되지 않는 경우 우선적으로 사용해야 합니다.

**특징:**
- **블록 스코프**: 블록 `{}` 내에서만 접근 가능
- **재할당 불가**: 선언 후 값을 변경할 수 없음
- **초기화 필수**: 선언할 때 반드시 값을 할당해야 함

**예제:**
```typescript
// 올바른 사용
const PI = 3.14159;
const userName = "Alice";
const isActive = true;

// ❌ 재할당 시도
PI = 3.14;  // TypeError: Assignment to constant variable

// ❌ 초기화 없이 선언
const name;  // SyntaxError: Missing initializer in const declaration
```

**중요: 객체와 배열의 const**

`const`는 변수의 재할당을 막을 뿐, **객체나 배열의 내용은 변경 가능**합니다:

```typescript
// 배열: 내용 수정은 가능
const numbers = [1, 2, 3];
numbers[0] = 10;    // ✅ OK (요소 수정)
numbers.push(4);    // ✅ OK (요소 추가)
numbers = [5, 6];   // ❌ 재할당 불가

// 객체: 속성 수정은 가능
const user = { name: "Alice", age: 25 };
user.name = "Bob";  // ✅ OK (속성 수정)
user = {};          // ❌ 재할당 불가

// 완전 불변 객체 만들기
const immutableUser = { name: "Alice" } as const;
immutableUser.name = "Bob";  // ❌ TypeError
```

---

### 1-2. let (값 변경 필요 시)

ES6에서 도입된 `let`은 **블록 스코프**를 가지며 **재할당이 가능**합니다. 값을 여러 번 변경해야 할 때 사용합니다.

**특징:**
- **블록 스코프**: 블록 내에서만 접근 가능
- **재할당 가능**: 값은 여러 번 변경 가능
- **재선언 불가**: 같은 스코프에서 중복 선언 불가

**예제:**
```typescript
// 값 변경 필요 시
let counter = 0;
counter = 1;  // ✅ OK
counter = 2;  // ✅ OK

// 루프 변수
for (let i = 0; i < 5; i++) {
  console.log(i);
}
console.log(i);  // ❌ ReferenceError (블록 스코프)

// ❌ 재선언 불가
let name = "Alice";
let name = "Bob";  // SyntaxError
```

---

### 1-3. var (권장하지 않음)

`var`는 ES5 이전의 변수 선언 방식으로, **현대 TypeScript에서는 거의 사용하지 않습니다**.

**문제점:**
- 함수 스코프만 지원 (블록 스코프 없음)
- 호이스팅으로 인한 예상치 못한 동작
- 재선언 가능으로 실수 유발

```typescript
// ❌ 권장하지 않음
var oldVariable = "outdated";
var oldVariable = "redeclared";  // 실수로 재선언됨
```

## 2. 기본 데이터 타입

TypeScript의 기본 데이터 타입은 JavaScript의 타입에 기반하며, 각각 고유한 특징을 가집니다.

### 2-1. string (문자열)

문자, 문자열을 나타냅니다.

```typescript
const name: string = "Alice";
const greeting: string = "Hello, TypeScript!";
const age: string = "25";  // 숫자처럼 보이지만 문자열

// 템플릿 리터럴
const message: string = `Hello, ${name}!`;  // "Hello, Alice!"

// ❌ 타입 오류
const invalid: string = 123;  // Type 'number' is not assignable to type 'string'
```

---

### 2-2. number (숫자)

정수, 소수, 음수, 특수 값(Infinity, NaN)을 나타냅니다.

```typescript
const age: number = 25;
const height: number = 5.9;
const temperature: number = -10;
const infinity: number = Infinity;
const notANumber: number = NaN;

// 다양한 진법
const binary: number = 0b1010;    // 이진수
const octal: number = 0o755;      // 팔진수
const hexadecimal: number = 0xFF;  // 십육진수

// ❌ 타입 오류
const invalid: number = "123";  // Type 'string' is not assignable to type 'number'
```

---

### 2-3. boolean (논리값)

`true` 또는 `false` 두 가지 값을 가집니다.

```typescript
const isActive: boolean = true;
const hasError: boolean = false;

// 조건문 결과
const isAdult: boolean = age >= 18;
const isEmpty: boolean = message.length === 0;

// ❌ 타입 오류
const invalid: boolean = 1;  // Type 'number' is not assignable to type 'boolean'
```

---

### 2-4. null과 undefined

특수한 값을 나타냅니다. 대부분 변수에 "값이 없음"을 표현합니다.

```typescript
// null: 명시적으로 "값이 없음"을 나타냄
const emptyValue: null = null;
let user: string | null = null;
user = "Alice";  // ✅ OK

// undefined: 아직 할당되지 않음
let uninitialized: undefined = undefined;
let response: string | undefined;
console.log(response);  // undefined

// 실제 사용 패턴
function findUser(id: number): string | null {
  if (id === 1) {
    return "Alice";
  }
  return null;  // 사용자를 찾지 못함
}

// optional 패턴
interface User {
  name: string;
  email?: string;  // undefined 가능
}
```

---

### 2-5. any (타입 검사 비활성화)

모든 타입을 허용합니다. **타입 안정성을 포기하므로 가능하면 피해야 합니다**.

```typescript
// ❌ 권장하지 않음
let anything: any = "string";
anything = 123;          // ✅ OK
anything = true;         // ✅ OK
anything.toUpperCase();  // ✅ OK (하지만 런타임 에러 가능)

// any는 타입 체킹을 완전히 우회합니다
const result: any = divideByZero();
const doubled = result * 2;  // 컴파일 에러 없음, 런타임 에러 가능
```

**any를 피해야 하는 이유:**
- 타입 안정성 상실
- IDE의 자동 완성 지원 불가
- 리팩토링 시 버그 발생 가능성

---

### 2-6. unknown (타입 검사 유지)

모든 타입을 허용하지만 **타입 검사를 유지합니다**. `any`보다 안전합니다.

```typescript
// ✅ 권장
let value: unknown = "string";
value = 123;          // ✅ OK
value = true;         // ✅ OK

// unknown은 타입 체크가 필수
// value.toUpperCase();  // ❌ Object is of type 'unknown'

// 타입 가드를 통한 사용
if (typeof value === "string") {
  console.log(value.toUpperCase());  // ✅ OK
}

if (typeof value === "number") {
  console.log(value * 2);  // ✅ OK
}

// 함수에서의 사용
function processData(data: unknown): void {
  if (typeof data === "string") {
    console.log(data.length);
  } else if (typeof data === "number") {
    console.log(data.toFixed(2));
  } else {
    console.log("Unknown type");
  }
}
```

## 3. 타입 어노테이션과 타입 추론

### 3-1. 타입 어노테이션 (명시적 지정)

변수를 선언할 때 콜론(`:`) 뒤에 타입을 명시합니다.

```typescript
// 변수 타입 지정
const name: string = "Alice";
const age: number = 25;
const isActive: boolean = true;

// 함수 매개변수와 반환 타입
function add(a: number, b: number): number {
  return a + b;
}

// 배열 타입
const numbers: number[] = [1, 2, 3];
const strings: Array<string> = ["a", "b", "c"];

// 객체 타입
interface User {
  name: string;
  age: number;
}
const user: User = { name: "Alice", age: 25 };
```

---

### 3-2. 타입 추론 (자동 감지)

타입을 명시하지 않아도 TypeScript가 할당된 값으로부터 타입을 자동으로 추론합니다.

```typescript
// 타입을 명시하지 않아도 자동 추론됨
const message = "Hello";      // 타입: string
const count = 0;              // 타입: number
const isValid = true;         // 타입: boolean

// 초기값이 없으면 any가 됨 (권장하지 않음)
let unknown;  // 타입: any

// 명시적으로 타입 지정 권장
let unknown: string | number;  // 처음부터 타입 정의

// 배열 타입 추론
const numbers = [1, 2, 3];    // 타입: number[]
const mixed = [1, "two"];     // 타입: (number | string)[]
```

---

### 3-3. Union 타입 (여러 타입 허용)

파이프(`|`)를 사용하여 여러 타입을 허용합니다.

```typescript
// 문자열 또는 숫자
let value: string | number = "Alice";
value = 25;           // ✅ OK
value = true;         // ❌ Type 'boolean' is not assignable

// 함수에서 사용
function processID(id: string | number): void {
  if (typeof id === "string") {
    console.log(id.toUpperCase());
  } else {
    console.log(id.toFixed(2));
  }
}

// 배열의 union 타입
const mixed: (string | number)[] = ["Alice", 25, "Bob", 30];
```

---

### 3-4. 타입 호환성 (Type Compatibility)

TypeScript는 **구조 기반 타입 호환성**(structural typing)을 사용합니다. 두 타입의 구조가 같으면 호환됩니다.

```typescript
interface Point {
  x: number;
  y: number;
}

// 명시적으로 Point 타입을 선언하지 않았지만 호환됨
const point = { x: 10, y: 20 };
const p: Point = point;  // ✅ OK (구조가 같음)

// 객체 리터럴 할당 시 엄격함
const invalid: Point = { x: 10, y: 20, z: 30 };  // ❌ 초과 속성

// 부분 호환도 안 됨
const incomplete: Point = { x: 10 };  // ❌ y 누락
```

## 4. 타입 체크 방법

### 4-1. typeof 연산자

런타임에 값의 타입을 확인합니다.

```typescript
const value: unknown = "hello";

if (typeof value === "string") {
  console.log(value.length);  // ✅ OK
}

// 지원되는 typeof 결과
typeof "string"     // "string"
typeof 123          // "number"
typeof true         // "boolean"
typeof undefined    // "undefined"
typeof null         // "object" (주의!)
typeof {}           // "object"
typeof []           // "object"
typeof function() {}  // "function"
```

## 5. 변수 네이밍 컨벤션

일관된 네이밍은 코드 가독성과 유지보수성을 높입니다.

### camelCase (변수, 함수, 메서드)

```typescript
const firstName = "Alice";
const getUserName = () => {};
const calculateTotal = (a: number, b: number) => a + b;
```

### PascalCase (클래스, 인터페이스, 타입)

```typescript
class UserManager {}
interface UserData {}
type UserInfo = {}
enum Status {}
```

### UPPER_SNAKE_CASE (상수)

```typescript
const MAX_USER_COUNT = 100;
const API_ENDPOINT = "https://api.example.com";
const DEFAULT_TIMEOUT = 5000;
```

### 언더스코어 접두사 (private 멤버 - 관례)

```typescript
class MyClass {
  _privateProperty: string = "private";
  
  constructor() {
    this._privateProperty = "initialized";
  }
}
```

## 6. 일반적인 패턴과 예제

### 6-1. 변수 선언 순서 (권장)

```typescript
// const를 기본값으로, 필요할 때만 let 사용
const MAX_RETRIES: number = 3;
const API_KEY: string = process.env.API_KEY || "";

let retryCount: number = 0;
let lastError: Error | null = null;

while (retryCount < MAX_RETRIES) {
  try {
    // API 호출
    break;
  } catch (error) {
    lastError = error as Error;
    retryCount++;
  }
}
```

### 6-2. Optional 변수

```typescript
interface Product {
  id: number;
  name: string;
  description?: string;  // optional
}

// 사용
const product: Product = {
  id: 1,
  name: "Laptop"
  // description은 생략 가능
};
```

### 6-3. 초기값 패턴

```typescript
// 기본값 설정
const config: { timeout: number; retries: number } = {
  timeout: 5000,
  retries: 3
};

// 객체 확산
const newConfig = { ...config, timeout: 10000 };
```
