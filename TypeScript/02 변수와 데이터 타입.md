# 변수 선언과 스코프

## 개요
TypeScript에서 변수를 선언하는 방식과 각각의 특징을 이해하는 것은 안정적인 코드를 작성하기 위한 기초입니다. `var`, `let`, `const` 세 가지 선언 방식의 차이와 스코프, 호이스팅을 명확하게 학습합니다.

---

## 1. 변수 선언 방식

### 1-1. var (구식, 권장하지 않음)
`var`는 ES5 이전의 변수 선언 방식으로, 현대 TypeScript에서는 거의 사용하지 않습니다.

**특징:**
- **함수 스코프**: 함수 내에서 선언된 var는 함수 전체에서 접근 가능
- **호이스팅**: 선언이 함수의 최상단으로 끌어올려짐 (undefined로 초기화)
- **재선언 가능**: 같은 이름의 변수를 여러 번 선언할 수 있음
- **재할당 가능**: 값을 여러 번 변경할 수 있음

**예제:**
```typescript
// 문제 1: 함수 스코프로 인한 예상치 못한 동작
function varExample() {
  if (true) {
    var x = 10;
  }
  console.log(x); // 10 (블록 스코프 무시)
}

// 문제 2: 호이스팅으로 인한 undefined
console.log(y); // undefined (에러가 아님!)
var y = 5;

// 문제 3: 재선언 가능
var name = "Alice";
var name = "Bob"; // 오류 없음
```

---

### 1-2. let (권장)
ES6에서 도입된 `let`은 블록 스코프를 가지며, 현대 TypeScript에서 일반적으로 사용됩니다.

**특징:**
- **블록 스코프**: 블록 `{}` 내에서만 접근 가능
- **호이스팅**: 선언되지만 초기화되지 않아 Temporal Dead Zone(TDZ) 발생
- **재선언 불가**: 같은 스코프에서 중복 선언 불가
- **재할당 가능**: 값은 여러 번 변경 가능

**예제:**
```typescript
// 올바른 블록 스코프
function letExample() {
  if (true) {
    let x = 10;
  }
  console.log(x); // ❌ ReferenceError: x is not defined
}

// 재할당은 가능
let count = 0;
count = 1; // ✅ OK
count = 2; // ✅ OK

// 재선언은 불가
let name = "Alice";
let name = "Bob"; // ❌ SyntaxError: Identifier 'name' has already been declared
```

**Temporal Dead Zone (TDZ):**
```typescript
console.log(age); // ❌ ReferenceError
let age = 25;

// TDZ는 블록 시작부터 선언까지의 영역
{
  console.log(value); // ❌ ReferenceError (TDZ)
  let value = 10;
  console.log(value); // ✅ 10
}
```

---

### 1-3. const (권장, 기본값)
ES6에서 도입된 `const`는 **재할당이 불가능한 변수**를 선언합니다. 값이 변경되지 않는 경우 우선적으로 사용해야 합니다.

**특징:**
- **블록 스코프**: `let`과 동일하게 블록 내에서만 접근 가능
- **호이스팅**: `let`과 동일하게 TDZ 발생
- **재선언 불가**: 같은 스코프에서 중복 선언 불가
- **재할당 불가**: 선언 후 값을 변경할 수 없음
- **초기화 필수**: 선언할 때 반드시 값을 할당해야 함

**예제:**
```typescript
// 올바른 사용
const PI = 3.14159;
console.log(PI); // ✅ 3.14159

// 재할당 시도
PI = 3.14; // ❌ TypeError: Assignment to constant variable

// 초기화 필수
const name; // ❌ SyntaxError: Missing initializer in const declaration
```

**중요: 객체와 배열의 const**

`const`는 변수 자체의 재할당을 막을 뿐, **객체나 배열의 내용은 변경 가능**합니다:

```typescript
// 배열 예제
const numbers = [1, 2, 3];
numbers[0] = 10; // ✅ OK (배열 내용 변경)
numbers.push(4); // ✅ OK (배열 요소 추가)
numbers = [5, 6]; // ❌ TypeError (재할당 불가)

// 객체 예제
const user = { name: "Alice", age: 25 };
user.name = "Bob"; // ✅ OK (객체 속성 변경)
user.age = 30; // ✅ OK (객체 속성 변경)
user = { name: "Charlie" }; // ❌ TypeError (재할당 불가)

// 불변 객체 만들기
const immutableUser = { name: "Alice" } as const;
immutableUser.name = "Bob"; // ❌ TypeError
```

## 2. 스코프 (Scope)

스코프는 변수에 접근할 수 있는 범위를 정의합니다.

### 2-1. 블록 스코프
`let`과 `const`는 블록 `{}`를 기준으로 스코프가 결정됩니다.

```typescript
{
  let blockVar = 10;
  const blockConst = 20;
}
console.log(blockVar); // ❌ ReferenceError
console.log(blockConst); // ❌ ReferenceError
```

### 2-2. 함수 스코프
함수 내에서 선언된 변수는 함수 내에서만 접근 가능합니다.

```typescript
function myFunction() {
  let localVar = 10;
  const localConst = 20;
  
  function innerFunction() {
    console.log(localVar); // ✅ 10 (외부 함수의 변수 접근 가능)
  }
  
  innerFunction();
}

console.log(localVar); // ❌ ReferenceError
```

### 2-3. 전역 스코프
함수나 블록 외부에서 선언된 변수는 전역 변수입니다.

```typescript
const globalVar = 10;

function myFunction() {
  console.log(globalVar); // ✅ 10
}

myFunction();
```

### 2-4. 클로저 (Closure)
함수가 외부 함수의 변수에 접근할 수 있는 메커니즘입니다.

```typescript
function outer() {
  const outerVar = 10;
  
  function inner() {
    const innerVar = 20;
    return outerVar + innerVar;
  }
  
  return inner;
}

const closure = outer();
console.log(closure()); // ✅ 30
```

## 3. 호이스팅 (Hoisting)

호이스팅은 변수 선언과 함수 선언이 스코프의 최상단으로 끌어올려지는 동작입니다.

### 3-1. var의 호이스팅
```typescript
console.log(x); // undefined (에러 아님)
var x = 10;
console.log(x); // 10

// 위 코드는 실제로 다음과 같이 동작합니다:
var x; // 호이스팅: 선언만 끌어올려짐
console.log(x); // undefined
x = 10; // 할당은 원래 위치에서 수행
console.log(x); // 10
```

### 3-2. let과 const의 호이스팅
```typescript
console.log(y); // ❌ ReferenceError: Cannot access 'y' before initialization
let y = 10;

// let과 const도 호이스팅되지만, TDZ(Temporal Dead Zone)에 의해
// 선언 전까지 접근할 수 없습니다.
```

## 4. TypeScript의 타입 어노테이션

TypeScript에서는 변수의 타입을 명시적으로 지정할 수 있습니다.

### 4-1. 기본 타입 어노테이션
```typescript
// 명시적 타입 지정
let name: string = "Alice";
let age: number = 25;
let isActive: boolean = true;

// 잘못된 타입 할당
name = 30; // ❌ Type 'number' is not assignable to type 'string'
```

### 4-2. 타입 추론 (Type Inference)
```typescript
// 타입을 명시하지 않아도 자동으로 추론됨
let message = "Hello"; // 타입: string
let count = 0; // 타입: number

message = 123; // ❌ Type 'number' is not assignable to type 'string'
```

### 4-3. union 타입
```typescript
// 여러 타입을 허용
let value: string | number = "Alice";
value = 25; // ✅ OK
value = true; // ❌ Type 'boolean' is not assignable to type 'string | number'
```

### 4-4. any와 unknown
```typescript
// any: 타입 검사 비활성화 (권장하지 않음)
let anyValue: any = "Alice";
anyValue = 25; // ✅ OK
anyValue.toUpperCase(); // ✅ OK (런타임 에러 발생 가능)

// unknown: 타입 검사 유지 (권장)
let unknownValue: unknown = "Alice";
unknownValue = 25; // ✅ OK
// unknownValue.toUpperCase(); // ❌ Object is of type 'unknown'
```

## 5. 변수 네이밍 컨벤션

일관된 네이밍은 코드 가독성과 유지보수성을 높입니다.

### 5-1. camelCase (변수와 함수)
```typescript
const firstName = "Alice"; // 변수
const calculateSum = () => {}; // 함수
let currentCount = 0;
```

### 5-2. PascalCase (클래스와 인터페이스)
```typescript
class UserManager {} // 클래스
interface UserData {} // 인터페이스
type UserType = {}; // 타입
```

### 5-3. UPPER_SNAKE_CASE (상수)
```typescript
const MAX_USER_COUNT = 100;
const API_ENDPOINT = "https://api.example.com";
const DEFAULT_TIMEOUT = 5000;
```

### 5-4. 언더스코어 규칙
```typescript
// private 멤버 (관례)
class MyClass {
  _privateProperty: string;
  
  constructor() {
    this._privateProperty = "private";
  }
}
```

## 6. var vs let vs const 선택 가이드

| 특징 | var | let | const |
|------|-----|-----|-------|
| 스코프 | 함수 | 블록 | 블록 |
| 재선언 | ✅ | ❌ | ❌ |
| 재할당 | ✅ | ✅ | ❌ |
| 호이스팅 | ✅ (undefined) | ✅ (TDZ) | ✅ (TDZ) |
| 권장도 | ❌ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

**선택 기준:**
1. **const를 기본값으로 사용**: 대부분의 경우 const 사용
2. **재할당이 필요한 경우만 let 사용**: 루프 변수, 상태 변경 등
3. **var는 절대 사용하지 않기**: 레거시 코드 유지보수 제외

---

## 요약

- **const**: 기본값, 값이 변경되지 않는 변수에 사용
- **let**: 값이 변경되는 변수에만 필요할 때 사용
- **var**: 현대 TypeScript에서 사용하지 않음
- **블록 스코프**: let과 const로 인한 안정적인 변수 관리
- **타입 어노테이션**: TypeScript의 강력한 기능으로 타입 안정성 확보
- **네이밍**: camelCase, PascalCase, UPPER_SNAKE_CASE 규칙 준수
